# 002_crawling 프로젝트

## 프로젝트 개요

OMDB API를 활용한 영화 정보 크롤링 및 저장 기능과, Google Gemini AI를 연동한 챗봇 기능을 제공하는 Django 웹 애플리케이션입니다.

## 주요 기능

### 1. 영화 검색 (`webcrawl` 앱)

| URL | 뷰 함수 | 설명 |
|-----|---------|------|
| `/webcrawl/` | `movie_search` | 검색 초기 화면 |
| `/webcrawl/all_search/` | `movie_all_search` | 2026년 영화 전체 조회 |
| `/webcrawl/qry_search/?qry=키워드` | `movie_qry_search` | 키워드로 영화 검색 |
| `/webcrawl/save_data/<imdbId>/` | `movie_save` | 선택한 영화를 DB에 저장 |

- OMDB API 호출로 영화 검색 결과 반환
- 저장 시 영화 제목을 자동으로 한국어로 번역하여 함께 저장
- `imdbID` 기준으로 중복 저장 방지

### 2. 영화 저장 (DB)

`Movie` 모델에 다음 정보를 저장합니다.

- 기본 정보: 제목(영문/한글), 연도, 등급, 개봉일, 상영시간, 장르
- 제작 정보: 감독, 작가, 배우, 줄거리, 언어, 국가, 수상
- 평가 정보: Metascore, IMDb Rating/Votes, Rotten Tomatoes 등
- 배급 정보: 매체, 박스오피스, 제작사, 웹사이트
- 포스터 URL, IMDB ID (unique)
- 생성/수정 타임스탬프

### 3. Gemini AI 챗봇 (`aichatbot` 앱)

- URL: `/aichatbot/`
- 사용자가 브라우저 `localStorage`에 Google Gemini API 키를 등록
- `gemini-2.5-flash` 모델 호출 (Axios 사용)
- 대화 이력을 `localStorage`에 최대 20개까지 저장/불러오기/삭제
- 마크다운(굵기, 기울임, 코드 블록) 간이 렌더링

### 4. 네이버 지식인 크롤링 (테스트 스크립트)

- `____test/naver_crawl.py`: 검색어 입력 → 네이버 지식인 결과 수집 → CSV 저장
- BeautifulSoup으로 HTML 파싱, 1~9페이지 반복 수집

### 5. 영어→한국어 자동 번역

- `webcrawl/translate.py`: Google Translate 비공개 API (`translate.googleapis.com`) 활용
- 외부 라이브러리 없이 표준 라이브러리(`urllib`)만 사용

## 동작 방법

### 1. 사전 준비

```bash
# 패키지 설치
pip install django pymysql requests beautifulsoup4 gradio

# MariaDB 데이터베이스 및 사용자 생성
CREATE DATABASE webcrawldb CHARACTER SET utf8mb4;
CREATE USER 'webcrawl'@'localhost' IDENTIFIED BY '1234';
GRANT ALL PRIVILEGES ON webcrawldb.* TO 'webcrawl'@'localhost';
```

### 2. DB 마이그레이션

```bash
python manage.py migrate
```

### 3. 서버 실행

```bash
python manage.py runserver
```

### 4. 접속

| 기능 | URL |
|------|-----|
| 영화 검색 | http://127.0.0.1:8000/webcrawl/ |
| AI 챗봇 | http://127.0.0.1:8000/aichatbot/ |
| 관리자 | http://127.0.0.1:8000/admin/ |

### 5. Gradio 챗봇 (독립 실행)

```bash
python ____gradio_chatbot/chatbot.py
```

## 설정 주요 사항

- `settings.py`의 `apiKey` 변수에 OMDB API 키 입력 필요 (`webcrawl/views.py` 내 빈 문자열)
- Gemini API 키는 클라이언트(브라우저 `localStorage`)에서 관리
- `ALLOWED_HOSTS = ['*']` — 개발 환경 전용, 운영 시 변경 필요
- `DEBUG = True` — 개발 환경 전용
